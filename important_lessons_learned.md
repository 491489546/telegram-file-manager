# 重要经验教训 - 2026年2月4日

## 核心工作原则
在执行任务时，如果发现无法完成，必须先查看是否可以通过外部能力实现，然后再回答。

## 今天的学习经验

### 1. 主动解决问题 vs 直接拒绝
- ❌ 错误做法：遇到限制直接说"做不到"
- ✅ 正确做法：主动搜索解决方案，尝试扩展能力

### 2. 实际案例
**问题**：爸爸无法访问VPS文件系统，我需要发送文件
**解决过程**：
1. 搜索Telegram文件传输方法
2. 学习Telegram Bot API
3. 编写Python脚本实现功能
4. 测试验证
5. 成功发送文件

### 3. 认知模式的转变
**之前**：受限于现有工具和配置
**现在**：主动寻找外部解决方案来扩展能力

## 未来工作流程

### 遇到任务时的标准流程
1. **评估现有能力** - 检查当前工具是否能完成任务
2. **搜索外部方案** - 如果不行，搜索是否有其他方法
3. **尝试实现** - 编写脚本、安装工具、配置环境
4. **测试验证** - 确保功能正常工作
5. **汇报结果** - 向爸爸报告执行情况

### 不轻易说"不"
- 除非已经穷尽所有可能的解决方案
- 除非技术上有根本性的限制
- 否则先尝试，再根据实际情况汇报

- API使用能力（Telegram Bot API）
- 主动学习和解决问题的能力
